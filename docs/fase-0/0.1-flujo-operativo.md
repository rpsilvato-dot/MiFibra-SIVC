# Documento de Cierre — FASE 0.1

# Proyecto: Sistema Inteligente de Validación Comercial MiFibra

---

# 1. Resumen Ejecutivo

La FASE 0.1 tuvo como objetivo documentar el flujo operativo REAL del proceso comercial de MiFibra, desde la captación del cliente hasta la aprobación, observación o rechazo de una venta.

El análisis permitió identificar que la operación actual depende fuertemente de:

- interpretación manual del Excel de condiciones comerciales
- validaciones dinámicas basadas en score/semaforización
- decisiones humanas operativas
- aprobaciones zonales
- revisión documental
- experiencia del supervisor

El sistema actual (“Scorts”) ya permite:

- registro de ventas
- consulta de score
- precalificación
- envío a validación
- observaciones
- reenvío

Sin embargo, no automatiza:

- interpretación de reglas comerciales
- requisitos dinámicos
- excepciones
- validaciones contextuales
- prevención de observaciones
- orientación operativa al asesor

El principal cuello de botella identificado es la dependencia operativa del supervisor como “intérprete humano” de las reglas comerciales.

La oportunidad del proyecto consiste en construir una capa inteligente que centralice y automatice la lógica del Excel de condiciones comerciales.

# 2. Flujo Operativo Completo

## Etapa 1 — Captación Comercial

### Actor principal:

- Asesor de ventas

### Actividades:

- Captación del cliente
- Explicación de planes
- Consulta inicial de cobertura
- Solicitud de datos básicos

### Validaciones iniciales:

- Cobertura
- DNI
- Score
- Semaforización

---

## Etapa 2 — Prevalidación Operativa

### Actor principal:

- Asesor
- Supervisor

### Actividades:

- Consulta del score mediante “lupita”
- Interpretación de restricciones
- Validación de elegibilidad
- Verificación de plan permitido
- Verificación de instalación
- Validación de campañas

### Decisiones:

- Venta directa
- Venta con sustento
- Venta con aprobación
- Venta no viable

---

## Etapa 3 — Recolección de Evidencias

### Evidencias principales:

- DNI ambas caras
- Fachada
- Contrato
- Router
- Selfie cliente
- Recibos
- Capturas de aprobación
- Audios

### Dependencia:

Las evidencias varían según:

- score
- semáforo
- tipo de venta
- campaña
- restricciones

---

## Etapa 4 — Solicitud de Aprobaciones

### Actor principal:

- Supervisor
- Jefe zonal

### Flujo:

- Supervisor interpreta Excel
- Genera formato de aprobación
- Adjunta evidencias
- Envía solicitud
- Jefe zonal responde

### Resultado:

- aprobado
- observado
- pendiente

---

## Etapa 5 — Registro en Scorts

### Flujo:

1. Login
2. Selección filial
3. Crear venta
4. Ingreso de datos
5. Consulta score
6. Adjuntar evidencias
7. Precalificación
8. Guardar o enviar

---

## Etapa 6 — Validación

### Actor:

- Área de validación

### Función:

- Verificación documental
- Verificación de reglas
- Verificación de aprobaciones
- Verificación de coincidencias

### Resultado:

- aprobado
- observado
- rechazado

---

## Etapa 7 — Subsanación

### Flujo:

- Venta observada
- Corrección
- Reenvío
- Nueva revisión

# Conclusión General

La operación comercial de MiFibra ya posee implícitamente un motor de reglas manual basado en:

- score
- semaforización
- restricciones dinámicas
- evidencias
- aprobaciones

El proyecto debe enfocarse en convertir esta lógica humana distribuida en:

- reglas estructuradas
- validaciones automatizadas
- recomendaciones operativas
- prevención de errores
- centralización de conocimiento comercial

sin reemplazar el flujo operativo principal existente.
